06_KIRA_Einstellungen_Design

Create a premium enterprise settings module UI for the same Kira product system and the same visual family as the already created dashboard, communication, business, knowledge, and Kira workspace reference pages.

Important:
This must clearly look like the same product family as the existing Kira pages.
Use the same elegant light SaaS app shell, the same collapsible left sidebar, the same top utility bar, the same refined neutral palette, the same typography feel, the same spacing rhythm, and the same premium enterprise product design language.

This is the “Einstellungen” module of the same Kira software.

Page layout:
full-width desktop application,
same left collapsible navigation,
same top utility bar,
below that a clear settings module header with:
title “Einstellungen”,
small live status row such as active provider, theme mode, notifications state, last change,
and utility actions on the right:
search, reset, export, import, save changes.

Main layout:
left side = section navigation for settings categories,
center = main settings configuration area,
right side optional = contextual help, preview, status, test actions.

Settings categories in the left settings navigation:
Design,
Benachrichtigungen,
Aufgabenlogik,
Nachfass-Intervalle,
Dashboard,
Kira / LLM / Provider,
Integrationen,
Automationen (planned badge).

Design section:
should feel like a premium design control center, not a raw form.
Show elegant grouped controls for:
theme mode,
accent color,
font size,
density,
company name,
logo upload,
card radius,
shadow strength,
reduce animations,
high contrast.
Include a clean live preview area that visually suggests these settings affect the interface.

Notifications section:
show structured switches and grouped notification preferences.
Should feel modern and easy to understand, not technical.

Task logic and follow-up sections:
show elegant control groups, intervals, priorities, timing settings, but in the same premium visual system.

Dashboard section:
show settings for KPI visibility, start view, compact vs. relaxed density, business signal visibility.
This should visually imply that the dashboard can be customized.

Kira / LLM / Provider section:
this is important and should feel like a refined mini console.
Show:
provider list,
fallback order,
model selection,
API status,
add provider,
test provider,
set default,
save fallback order.
Make it look like a polished enterprise AI provider management panel, not a developer-only JSON page.

Integrations section:
show current and planned integrations in a structured way.
Mail, OneDrive / Dokumente, Push should look active-capable.
Calendar and Automations should have subtle “In Planung” badges.

Visual direction:
must feel like premium 2026 business software,
clean, calm, highly readable,
larger text,
clear groups,
strong visual hierarchy,
same design DNA as the rest of the Kira product.

Important constraints:
no raw long forms,
no tiny inputs,
no weak contrast,
no pale low-impact UI,
no generic admin settings page,
no rauMKult brand colors,
no hardcoded branding,
no dark mode,
no retro interface,
no old Windows control panel look.

Use:
crisp white and soft neutral surfaces,
dark readable text,
subtle restrained accent colors only for active states,
premium spacing,
refined shadows,
clear grouping,
elegant toggles, segmented controls, dropdowns, and configuration cards.

Allowed:
same subtle Kira launcher button in the bottom right corner, closed state only.

Style keywords:
premium SaaS settings module, enterprise configuration console, elegant admin preferences UI, provider management panel, refined design controls, modern business software settings, same visual system as the Kira reference modules.
Avoid: raw settings form, developer-only config page, generic admin panel, old Windows control panel, retro UI, tiny text, weak contrast, washed out colors, narrow center column, dark cyber style, cluttered controls, mismatched design language, generic CMS settings page.
